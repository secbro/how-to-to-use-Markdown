怎样使用Markdown

需要更多信息的同学,可以查阅:英文原文

Markdown是一款基于Github平台的、简单易用的规范文章格式的语法。

你能从这篇文章学到什么：

  • 使用Markdown转变传统样式编辑的文本
  • 使用Markdown进行文本编辑
  • 应用Markdown的个性扩展

什么是Markdown？

Markdown是一种在web中编辑文本样式的方式。你可以利用Markdown控制文件的显示方式；对文字进行加粗或使其倾斜，添加图片，创造列表······简单来说，Markdown就是一段普通的文本加上一些特殊符号，比如#,*。

你可以使用Markdown的地方包括但不限于：

  • Gists
  • Pull Requests 和 Issues的时候
  • 以.md和.markdown为扩展名的文件中

例子

Text

It's very easy to make some words **bold** and other words *italic* with 
Markdown. You can even [link to Google!](http://google.com).

It's very easy to make some words bold and other words italic with Markdown. You can even link to Google!.

语法导引

下面的语法你可以在Github的任意地方使用

Headers

语法格式

# This is an h1 tag
## This is an h2 tag
###### This is an h6 tag

显示结果

This is an h1 tag

This is an h2 tag

This is an h6 tag

Emphasis

语法格式

*This text will be italic*
_This will also be italic_

**This text will be bold**
__This will also be bold__

*You **can** combine them*

显示结果

This text will be italic

This will also be italic

This text will be bold

This will also be bold

You can combine them

Lists

无序

语法格式

* Item 1
* Item 2
  * Item 2a
  * Item 2b

显示结果

  • Item 1
  • Item 2
    • Item 2a
    • Item 2b

有序

语法格式

1. Item 1
2. Item 2
3. Item 3
  * Item 3a
  * Item 3b

显示结果

  1. Item 1
  2. Item 2
  3. Item 3
    • Item 3a
    • Item 3b

链接

语法格式

http://github.com - automatic!
[GitHub](http://github.com)

显示结果

http://github.com - automatic! GitHub

GitHub Flavored Markdown

Github使用了Markdown的独特版本,这使它具备了一些特性.

其中的一些特性只在只有在Issues和Pull Requests的描述和评论中有效.

高亮文字

语法格式

```javascript
function fancyAlert(arg) {
  if(arg) {
    $.facebox({div:'#foo'})
  }
}
```

显示结果

function fancyAlert(arg) {
  if(arg) {
    $.facebox({div:'#foo'})
  }
}

表格

语法格式

First Header | Second Header
------------ | -------------
Content from cell 1 | Content from cell 2
Content in the first column | Content in the second column

显示结果

First Header Second Header
Content from cell 1 Content from cell 2
Content in the first column Content in the second column
总结说来,Markdown用起来简单,但是要注意空格和分行,特别是在与html代码结合使用的时候.

我在上面的例子中,使用pre标签包裹Markdown的时候没有换行,出了不少错误.在使用Markdown的时候,要特别注意换行问题!
